Overview

SONY가 가지고 있는 세련된 디자인과 스타일리시함을 보여주고자 하였습니다.

또한 미디어 쿼리를 활용하여 다양한 디바이스에도 잘 작동할 수 있는
반응형 웹사이트를 만드는 것을 목표로 삼았습니다.

  • 세련된 디자인
  • 반응형 웹사이트
  • MediaQuery 활용

Design

Main Color

SONY의 세련됨과 깔끔함의 이미지를 살려 블랙 & 화이트를 사용하였습니다.

Sub Color

메인 컬러를 헤치지않는 채도가 낮은 블루계열의 컬러를 사용하였습니다.

Font

  • MADE Outer Sans (영문)
  • Bebas Neue (영문)

강조를 위한 영문은 MADE Outer Sans을 사용하고,
소니의 세련된 이미지에 맞게 Neue Montreal를 사용했습니다.

Publishing

  • JavaScript를 활용한 다양한 이벤트 효과 구현
  • canvas를 사용한 애니메이션 이미지
  • grid를 사용하여 반응형에 알맞는 레이아웃 구현
  • gsap의 toArray 사용한 텍스트 효과
  • Swiper를 적용한 여러 슬라이드

Problems
& Solution

  • 슬라이드, 반응형에 왜 반응을 안할까?
  • pc 버전에서는 슬라이드의 레이아웃이 제대로 이루어져있었으나
    viewport의 크기가 줄어들면 레이아웃이 무너지는 현상이 발생하였습니다.
  • 슬라이드를 만들었던 Swiper 코드에 breakpoints를 활용하였고,
    레이아웃이 무너지는 viewport의 크기를 확인 후 slidesPerView 조절하여
    반응형에도 제대로 보여질 수 있도록 하였습니다.
  • canvas의 width 값, 왜 새로고침을 해야 돌아올까?
  • line-bg 영역 내에 스크롤이 왔을 시 라인 3개가 움직이는 이벤트를 적용하였으나
    스크롤이 영역 밖으로 갔다가 다시 돌아오면 이벤트가 다시 발생하지 않았습니다.
  • JavaScript에 조건문을 추가하여
    if - 영역 내에 있을 때는 class가 적용되고 else - 영역 밖에 있을 때 class가
    적용되지 않게 코드를 변경하였더니 해결되었습니다.

Review

반응형 웹사이트를 목표로 시작한 만큼, 디자인부터 퍼블리싱 구조까지 신경 써야 하는 부분이 많았던 것 같습니다.

veiwport에 따라 계속해서 변하는 레이아웃으로 인한 여러 번의 코드 수정은 저를 힘들게 하기도 하였습니다.
그러나 다양한 디바이스에서 내가 만든 사이트를 볼 수 있다는 점은 오히려 저에게 해낼 수 있다는 힘을 가져다 주기도 했습니다.

반응형 웹사이트는 저에게 두려움이었지만,
이번 개인 프로젝트를 완성했을 때의 큰 뿌듯함은 그 두려움의 크기를 작게 줄여준 계기가 되었습니다.
반응형 웹사이트를 만드는 과정 속에서 얻은 경험과 지식은 앞으로의 프로젝트에서도 큰 도움이 될 것입니다.
